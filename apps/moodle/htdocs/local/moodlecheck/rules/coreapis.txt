<h2><span class="mw-headline" id="Most-used_General_APIs">Most-used General APIs</span></h2>
<p>These APIs are critical and will be used by nearly every Moodle plugin.
</p>
<h3><span class="mw-headline" id="Access_API_.28access.29">Access API (access)</span></h3>
<p>The <a href="/dev/Access_API" title="Access API">Access API</a> gives you functions so you can determine what the current user is allowed to do, and it allows modules to extend Moodle with new capabilities.
</p>
<h3><span class="mw-headline" id="Data_manipulation_API_.28dml.29">Data manipulation API (dml)</span></h3>
<p>The <a href="/dev/Data_manipulation_API" title="Data manipulation API">Data manipulation API</a> allows you to read/write to databases in a consistent and safe way.
</p>
<h3><span class="mw-headline" id="File_API_.28files.29">File API (files)</span></h3>
<p>The <a href="/dev/File_API" title="File API">File API</a> controls the storage of files in connection to various plugins.
</p>
<h3><span class="mw-headline" id="Form_API_.28form.29">Form API (form)</span></h3>
<p>The <a href="/dev/Form_API" title="Form API">Form API</a> defines and handles user data via web forms.
</p>
<h3><span class="mw-headline" id="Logging_API_.28log.29">Logging API (log)</span></h3>
<p>The <a href="/dev/Events_API" title="Events API">Events API</a> allows you to log events in Moodle, while <a href="/dev/Logging_2" title="Logging 2">Logging 2</a> describes how logs are stored and retrieved.
</p>
<h3><span class="mw-headline" id="Navigation_API_.28navigation.29">Navigation API (navigation)</span></h3>
<p>The <a href="/dev/Navigation_API" title="Navigation API">Navigation API</a> allows you to manipulate the navigation tree to add and remove items as you wish.
</p>
<h3><span class="mw-headline" id="Page_API_.28page.29">Page API (page)</span></h3>
<p>The <a href="/dev/Page_API" title="Page API">Page API</a> is used to set up the current page, add JavaScript, and configure how things will be displayed to the user.
</p>
<h3><span class="mw-headline" id="Output_API_.28output.29">Output API (output)</span></h3>
<p>The <a href="/dev/Output_API" title="Output API">Output API</a> is used to render the HTML for all parts of the page.
</p>
<h3><span class="mw-headline" id="String_API_.28string.29">String API (string)</span></h3>
<p>The <a href="/dev/String_API" title="String API">String API</a> is how you get language text strings to use in the user interface.   It handles any language translations that might be available.
</p>
<h3><span class="mw-headline" id="Upgrade_API_.28upgrade.29">Upgrade API (upgrade)</span></h3>
<p>The <a href="/dev/Upgrade_API" title="Upgrade API">Upgrade API</a> is how your module installs and upgrades itself, by keeping track of its own version.
</p>
<h3><span class="mw-headline" id="Moodlelib_API_.28core.29">Moodlelib API (core)</span></h3>
<p>The <a href="/dev/index.php?title=Moodlelib_API&amp;action=edit&amp;redlink=1" class="new" title="Moodlelib API (page does not exist)">Moodlelib API</a> is the central library file of miscellaneous general-purpose Moodle functions. Functions can over the handling of request parameters, configs, user preferences, time, login, mnet, plugins, strings and others. There are plenty of defined constants too.
</p>
<h2><span class="mw-headline" id="Other_General_APIs">Other General APIs</span></h2>
<h3><span class="mw-headline" id="Admin_settings_API_.28admin.29">Admin settings API (admin)</span></h3>
<p>The <a href="/dev/Admin_settings" title="Admin settings">Admin settings</a> API deals with providing configuration options for each plugin and Moodle core.
</p>
<h3><span class="mw-headline" id="Analytics_API_.28analytics.29">Analytics API (analytics)</span></h3>
<p>The <a href="/dev/Analytics_API" title="Analytics API">Analytics API</a> allow you to create prediction models and generate insights.
</p>
<h3><span class="mw-headline" id="Availability_API_.28availability.29">Availability API (availability)</span></h3>
<p>The <a href="/dev/Availability_API" title="Availability API">Availability API</a> controls access to activities and sections.
</p>
<h3><span class="mw-headline" id="Backup_API_.28backup.29">Backup API (backup)</span></h3>
<p>The <a href="/dev/Backup_API" title="Backup API">Backup API</a> defines exactly how to convert course data into XML for backup purposes, and the <a href="/dev/Restore_API" title="Restore API">Restore API</a> describes how to convert it back the other way.
</p>
<h3><span class="mw-headline" id="Cache_API_.28cache.29">Cache API (cache)</span></h3>
<p>The <a href="/dev/The_Moodle_Universal_Cache_(MUC)" title="The Moodle Universal Cache (MUC)">The Moodle Universal Cache (MUC)</a> is the structure for storing cache data within Moodle. <a href="/dev/Cache_API" title="Cache API">Cache_API</a> explains some of what is needed to use a cache in your code.
</p>
<h3><span class="mw-headline" id="Calendar_API_.28calendar.29">Calendar API (calendar)</span></h3>
<p>The <a href="/dev/Calendar_API" title="Calendar API">Calendar API</a> allows you to add and modify events in the calendar for user, groups, courses, or the whole site.
</p>
<h3><span class="mw-headline" id="Check_API_.28check.29">Check API (check)</span></h3>
<p>The <a href="/dev/Check_API" title="Check API">Check API</a> allows you to add security, performance or health checks to your site.
</p>
<h3><span class="mw-headline" id="Comment_API_.28comment.29">Comment API (comment)</span></h3>
<p>The <a href="/dev/Comment_API" title="Comment API">Comment API</a> allows you to save and retrieve user comments, so that you can easily add commenting to any of your code.
</p>
<h3><span class="mw-headline" id="Competency_API_.28competency.29">Competency API (competency)</span></h3>
<p>The <a href="/dev/Competency_API" title="Competency API">Competency API</a> allows you to list and add evidence of competencies to learning plans, learning plan templates, frameworks, courses and activities.
</p>
<h3><span class="mw-headline" id="Data_definition_API_.28ddl.29">Data definition API (ddl)</span></h3>
<p>The <a href="/dev/Data_definition_API" title="Data definition API">Data definition API</a> is what you use to create, change and delete tables and fields in the database during upgrades.
</p>
<h3><span class="mw-headline" id="Editor_API">Editor API</span></h3>
<p>The <a href="/dev/Editor_API" title="Editor API">Editor API</a> is used to control HTML text editors.
</p>
<h3><span class="mw-headline" id="Enrolment_API_.28enrol.29">Enrolment API (enrol)</span></h3>
<p>The <a href="/dev/Enrolment_API" title="Enrolment API">Enrolment API</a> deals with course participants.
</p>
<h3><span class="mw-headline" id="Events_API_.28event.29">Events API (event)</span></h3>
<p>The <a href="/dev/Events_API" title="Events API">Events API</a> allows to define "events" with payload data to be fired whenever you like, and it also allows you to define handlers to react to these events when they happen.  This is the recommended form of inter-plugin communication. This also forms the basis for logging in Moodle.
</p>
<h3><span class="mw-headline" id="Experience_API_.28xAPI.29">Experience API (xAPI)</span></h3>
<p>The Experience API (xAPI) is an e-learning standard that allows learning content and learning systems to speak to each other. The <a href="/dev/Experience_API_(xAPI)" title="Experience API (xAPI)">Experience API (xAPI)</a>
allows any plugin to generate and handle xAPI standard statements.
</p>
<h3><span class="mw-headline" id="External_functions_API_.28external.29">External functions API (external)</span></h3>
<p>The <a href="/dev/External_functions_API" title="External functions API">External functions API</a> allows you to create fully parametrised methods that can be accessed by external programs (such as <a href="/dev/Web_services" title="Web services">Web services</a>).
</p>
<h3><span class="mw-headline" id="Favourites_API">Favourites API</span></h3>
<p>The <a href="/dev/Favourites_API" title="Favourites API">Favourites API</a> allows you to mark items as favourites for a user and manage these favourites. This is often referred to as 'Starred'.
</p>
<h3><span class="mw-headline" id="Lock_API_.28lock.29">Lock API (lock)</span></h3>
<p>The <a href="/dev/Lock_API" title="Lock API">Lock API</a> lets you synchronise processing between multiple requests, even for separate nodes in a cluster.
</p>
<h3><span class="mw-headline" id="Message_API_.28message.29">Message API (message)</span></h3>
<p>The <a href="/dev/Message_API" title="Message API">Message API</a> lets you post messages to users.  They decide how they want to receive them.
</p>
<h3><span class="mw-headline" id="Media_API_.28media.29">Media API (media)</span></h3>
<p>The <a href="/dev/Media_players#Using_media_players" title="Media players">Media</a> API can be used to embed media items such as audio, video, and Flash.
</p>
<h3><span class="mw-headline" id="My_profile_API">My profile API</span></h3>
<p>The <a href="/dev/My_profile_API" title="My profile API">My profile API</a> is used to add things to the profile page.
</p>
<h3><span class="mw-headline" id="OAuth_2_API_.28oauth2.29">OAuth 2 API (oauth2)</span></h3>
<p>The <a href="/dev/OAuth_2_API" title="OAuth 2 API">OAuth 2 API</a> is used to provide a common place to configure and manage external systems using OAuth 2.
</p>
<h3><span class="mw-headline" id="Preference_API_.28preference.29">Preference API (preference)</span></h3>
<p>The <a href="/dev/Preference_API" title="Preference API">Preference API</a> is a simple way to store and retrieve preferences for individual users.
</p>
<h3><span class="mw-headline" id="Portfolio_API_.28portfolio.29">Portfolio API (portfolio)</span></h3>
<p>The <a href="/dev/Portfolio_API" title="Portfolio API">Portfolio API</a> allows you to add portfolio interfaces on your pages and allows users to package up data to send to their portfolios.
</p>
<h3><span class="mw-headline" id="Privacy_API_.28privacy.29">Privacy API (privacy)</span></h3>
<p>The <a href="/dev/Privacy_API" title="Privacy API">Privacy API</a> allows you to describe the personal data that you store, and provides the means for that data to be discovered, exported, and deleted on a per-user basis.
This allows compliance with regulation such as the General Data Protection Regulation (GDPR) in Europe.
</p>
<h3><span class="mw-headline" id="Rating_API_.28rating.29">Rating API (rating)</span></h3>
<p>The <a href="/dev/Rating_API" title="Rating API">Rating API</a> lets you create AJAX rating interfaces so that users can rate items in your plugin.  In an activity module, you may choose to aggregate ratings to form grades.
</p>
<h3><span class="mw-headline" id="RSS_API_.28rss.29">RSS API (rss)</span></h3>
<p>The <a href="/dev/RSS_API" title="RSS API">RSS API</a> allows you to create secure RSS feeds of data in your module.
</p>
<h3><span class="mw-headline" id="Search_API_.28search.29">Search API (search)</span></h3>
<p>The <a href="/dev/Search_API" title="Search API">Search API</a> allows you to index contents in a search engine and query the search engine for results.
</p>
<h3><span class="mw-headline" id="Tag_API_.28tag.29">Tag API (tag)</span></h3>
<p>The <a href="/dev/Tag_API" title="Tag API">Tag API</a> allows you to store tags (and a tag cloud) to items in your module.
</p>
<h3><span class="mw-headline" id="Task_API_.28task.29">Task API (task)</span></h3>
<p>The <a href="/dev/Task_API" title="Task API">Task API</a> lets you run jobs in the background. Either once off, or on a regular schedule.
</p>
<h3><span class="mw-headline" id="Time_API_.28time.29">Time API (time)</span></h3>
<p>The <a href="/dev/Time_API" title="Time API">Time API</a> takes care of translating and displaying times between users in the site.
</p>
<h3><span class="mw-headline" id="Testing_API_.28test.29">Testing API (test)</span></h3>
<p>The testing API contains the Unit test API (<a href="/dev/PHPUnit" title="PHPUnit">PHPUnit</a>) and Acceptance test API (<a href="/dev/Acceptance_testing" title="Acceptance testing">Acceptance testing</a>). Ideally all new code should have unit tests written FIRST.
</p>
<h3><span class="mw-headline" id="User-related_APIs_.28user.29">User-related APIs (user)</span></h3>
<p>This is a rather informal grouping of miscellaneous <a href="/dev/User-related_APIs" title="User-related APIs">User-related APIs</a> relating to sorting and searching lists of users.
</p>
<h3><span class="mw-headline" id="Web_services_API_.28webservice.29">Web services API (webservice)</span></h3>
<p>The <a href="/dev/Web_services_API" title="Web services API">Web services API</a> allows you to expose particular functions (usually external functions) as web services.
</p>
<h3><span class="mw-headline" id="Badges_API_.28badges.29">Badges API (badges)</span></h3>
<p>The <a rel="nofollow" class="external text" href="https://docs.moodle.org/dev/OpenBadges_User_Documentation">Badges</a> user documentation (is a temp page until we compile a proper page with all the classes and APIs that allows you to manage particular badges and OpenBadges Backpack).
</p>
<h3><span class="mw-headline" id="Custom_fields_API">Custom fields API</span></h3>
<p>The <a href="/dev/Custom_fields_API" title="Custom fields API">Custom fields API</a> allows you to configure and add custom fields for different entities
</p>
<h2><span class="mw-headline" id="Activity_module_APIs">Activity module APIs</span></h2>
<p>Activity modules are the most important plugin in Moodle.  There are several core APIs that service only Activity modules.
</p>
<h3><span class="mw-headline" id="Activity_completion_API_.28completion.29">Activity completion API (completion)</span></h3>
<p>The <a href="/dev/Activity_completion_API" title="Activity completion API">Activity completion API</a> is to indicate to the system how activities are completed.
</p>
<h3><span class="mw-headline" id="Advanced_grading_API_.28grading.29">Advanced grading API (grading)</span></h3>
<p>The <a href="/dev/Advanced_grading_API" title="Advanced grading API">Advanced grading API</a> allows you to add more advanced grading interfaces (such as rubrics) that can produce simple grades for the gradebook.
</p>
<h3><span class="mw-headline" id="Conditional_activities_API_.28condition.29_-_deprecated_in_2.7">Conditional activities API (condition) - deprecated in 2.7</span></h3>
<p>The deprecated <a href="/dev/Conditional_activities_API" title="Conditional activities API">Conditional activities API</a> used to provide conditional access to modules and sections in Moodle 2.6 and below. It has been replaced by the <a href="/dev/Availability_API" title="Availability API">Availability API</a>.
</p>
<h3><span class="mw-headline" id="Groups_API_.28group.29">Groups API (group)</span></h3>
<p>The <a href="/dev/Groups_API" title="Groups API">Groups API</a> allows you to check the current activity group mode and set the current group.
</p>
<h3><span class="mw-headline" id="Gradebook_API_.28grade.29">Gradebook API (grade)</span></h3>
<p>The <a href="/dev/Gradebook_API" title="Gradebook API">Gradebook API</a> allows you to read and write from the gradebook.  It also allows you to provide an interface for detailed grading information.
</p>
<h3><span class="mw-headline" id="Plagiarism_API_.28plagiarism.29">Plagiarism API (plagiarism)</span></h3>
<p>The <a href="/dev/Plagiarism_API" title="Plagiarism API">Plagiarism API</a> allows your activity module to send files and data to external services to have them checked for plagiarism.
</p>
<h3><span class="mw-headline" id="Question_API_.28question.29">Question API (question)</span></h3>
<p>The <a href="/dev/Question_API" title="Question API">Question API</a> (which can be divided into the Question bank API and the Question engine API), can be used by activities that want to use questions from the question bank.
</p>
